# Code Review Checklist

Use this checklist when reviewing pull requests to ensure consistent quality.

## Before You Start

- [ ] Read the PR description and linked issue
- [ ] Understand the context and goals
- [ ] Check the size - large PRs should be split

## Code Quality

1. **Readability**: Is the code easy to understand?
2. **Naming**: Are variables, functions, and classes named clearly?
3. **Structure**: Is the code well-organized?
